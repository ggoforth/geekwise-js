Operators

We talked a bit about number data types in the last class, but not much about what we can do with them.
Operators allow us to take action and do something with the variables we've created.
Some operators you will see often include:
  • Mathmatical
    • = (equals, assignment)
    • + (addition)
    • - (subtraction)
    • * (multiplication)
    • / (division)
    • % (modulus)

    • Shorthand Operators

    • +=
    • -=
    • *=
    • /=
  • Comparison
    • == (loose comparison)
    • === (strict comparison)
    • != (loose not equal comparison)
    • !== (strict not equal comparison)
    • < (less than)
    • > (greater than)
    • <= (less than or equal to)
    • >= (greater than or equal to)
  • Logical
    • && (and)
    • || (or)

Math

You can perform mathmatical calculations using operators as discussed above.
Try it: open your console and run a few calculations like 2 + 2, etc...
The "value" of the expression can be stored for later use.
Try it: create a variable called result. Add two numbers together, and store the value in your result variable using the = operator. Log that value to the console.
Shorthand notation exists for quickly operating on an existing variable.
  • +=
  • -=
  • *=
  • /=
With these shortcut helpers, you can turn this: 

var a = 0;
a = a + 5; //5

var b = 10;
b = b / 5; //2
into this: 
    
var a = 0;

a += 5;

console.log(a);


var b = 10;

b /= 5;

console.log(b);
Operator Precedence, works like it did when you were in sixth grade.
  1. Parentheses
  2. Multiplication / Division
  3. Addition / Subtraction

//what's the answer?
var a = 5 + 5 * 10;
console.log(a);
If you wanted to have the 5 + 5 evaluate first, you would need to use paren's to group the parts you want to take precedence. 

//what's the answer?
var a = (5 + 5) * 10;
console.log(a);
The modulus % is a usful operator to know how to leverage.
You can use modulus to determine if something is even or odd, a typical use is if you are generating content like rows in a table, and you want to put an "even" class on even rows and and "odd" class on odd rows.

var a = 5;
console.log(a % 2); // 1


var b = 6;
console.log(b % 3); // 0
It's also very easy to increment / decrement a number by one using javascript.
Use the ++ or -- operators, Note: these do not require re-assigning the value to a variable, the work on existing variables.

var a = 5;
a++;
console.log(a);


var b = 6;
b--;
console.log(b);
parseInt(number) can be used for converting a float to an int, or for turning a string into an int.

var a = 1.6;
var b = parseInt(a); // number data type 1

var c = "5";
var d = parseInt(c); // number data type 5
If you try and parse a value that can't be turned into a number, you will get NaN ( not a number)

var a = "foo";
var b = parseInt(a); // NaN
You can check if a value is NaN by using the isNaN function. 

var a = "foo";
var b = parseInt(a); //NaN
console.log(isNaN(b)); //true

Control Flow

Control flow simply means controlling what code actually gets executed in your program.
It's very common that your application will want to behave differently based on available data. One common scenario is a login / authentication strategy. If you build an application where you want to require the users logs in, you'll want to show a login screen if they aren't logged in, or redirect them to their home page if they are.
The kinds of control flow we are going to look at today:
  • if
  • if / else
  • if / elseif
  • if / elseif / else
  • switch
  • ternary
In almost all cases, the control flow will evaluate some condition, and react accordingly. This enables us to run some code, while skipping other parts.
The most basic form of control flow are if / else statements.
    
//simple if statement
if (true) {

    console.log("yup");

}

//if / else, if the first condition is true, run the first
//block, else run the second block
if (false) {

    alert("it's true");

} else {

    alert("it's false");

}

//if else if statments, the last else is our "fallback"
if (condition1) {


} else if (condition2) {


} else {


}
With if statements, it's possible to evaluate multiple conditions, and we use logical operators to do so. This lets us say "If this AND that is true, then run this code.
  • && the and operator
  • || the or operator

var a = true;
var b = false;

//if both a AND b are true...
if (a && b) {
    console.log("both a and b are truthy");
} else {
    console.log("one of a or b are false");
}

//if a OR b is true
if (a || b) {
    console.log("one of a or b are true");
}
Another common control flow element is the switch statement. Switch will evaluate the condition and then perform a given block of code based on that value. It looks like: 

var condition = "foo";

switch (condition) {
case "foo":
    alert("foo");
    break;
case "bar":
    alert("bar");
    break;
default:
    alert("wasn't foo or bar");
}
Careful with Switch statements however, they can lead to some unwanted results if you forget the break keyword called "fall-through" or "pass-through". What happens when you run the switch statement and omit the break in the "foo" case? 

var condition = "foo";

switch (condition) {
case "foo":
    alert("foo");
case "bar":
    alert("bar");
    break;
default:
    alert("wasn't foo or bar");
}
Ternary operators are a short hand if / else type of control flow.
They use the ? and the : as the syntax.
It looks like: condition ? trueValue : falseValue; 

var loggedIn = true;
var needsLogin = loggedIn ? false : true;

var age = 31;
var logout = age < 13 ? true : false;
Ternary operators are great for simply quick conditions where the evaluates are simple
Avoid nested ternary

var dogs = false
var cats = false;
var fish = true;
var hasPets = dogs ? true : cats ? true : fish ? true : false; //wat?
Just because you can do something, doesn't mean you should. Just because you have a hammer, not everything is a nail.
